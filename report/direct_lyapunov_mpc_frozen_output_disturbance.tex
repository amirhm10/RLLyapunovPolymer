\begin{abstract}
This report documents the direct Lyapunov MPC experiment in
\texttt{DirectLyapunovMPC\_FrozenOutputDisturbance.ipynb}. The method replaces
the earlier candidate-and-filter workflow with one controller solve at every
time step: freeze the output disturbance estimate, compute a steady target, and
solve a Lyapunov-constrained tracking MPC problem around that target. The
notebook has now been rewritten as a four-scenario study runner covering
\texttt{unbounded\_hard}, \texttt{bounded\_hard},
\texttt{unbounded\_soft}, and \texttt{bounded\_soft}. The report also records
the earlier diagnostic run from April 23, 2026, compares the idea with the
stronger but more heavily regularized
\texttt{LyapunovSafetyFilterMPCTargetSelectorTermAblation.ipynb} path, and
places the current design against offset-free MPC, tracking MPC, Lyapunov MPC,
and predictive safety-filter literature. The prior run is diagnostically useful:
the frozen target calculation succeeds exactly, but the \texttt{unbounded} target
combined with \texttt{hard} Lyapunov contraction makes the direct MPC infeasible
at all 1600 steps. The next experimental stage should therefore test bounded
targets and soft Lyapunov contraction in the four-scenario comparison before
adding any of the richer selector regularization terms back into the direct path.
\end{abstract}

\section{Executive Summary}

The direct frozen-output-disturbance Lyapunov MPC path is a cleaner controller
architecture for the polymer CSTR experiments. Instead of first producing an
upstream MPC or RL candidate and then using a safety filter to accept, project,
or replace that candidate, the direct path solves one Lyapunov MPC problem at
each online step. This makes the closed-loop law easier to explain and easier to
diagnose.

The main workflow is:
\begin{enumerate}[leftmargin=1.5em]
\item estimate the augmented observer state
\[
\hat z_k =
\begin{bmatrix}
\hat x_k \\
\hat d_k
\end{bmatrix},
\]
\item freeze the output disturbance target as \(d_s=\hat d_k\),
\item compute a steady target \((x_s,u_s,y_s)\),
\item solve a tracking MPC problem with terminal ingredients and a first-step
Lyapunov contraction constraint,
\item apply the first input and export target, solver, contraction, and plotting
diagnostics.
\end{enumerate}

The notebook now runs all four target/Lyapunov combinations by default and saves
per-case bundles plus a study-level comparison table. The earlier saved run used
\texttt{target\_mode="unbounded"} and \texttt{lyapunov\_mode="hard"}. It showed
a sharp separation between the two pieces of the method:
\begin{itemize}[leftmargin=1.5em]
\item the target solve is numerically successful, with maximum total residual
\(4.58\times 10^{-15}\),
\item the unbounded target is outside the admissible input box at every step,
\item the hard direct MPC solve is infeasible at every step,
\item the controller therefore holds the previous input for the whole run.
\end{itemize}

This means the report should not be read as evidence that the direct controller
is already tuned. It is evidence that the new instrumentation is doing its job:
it identifies the target admissibility and hard-contraction bottleneck cleanly.

\section{Why This Controller Exists}

The repository has several Lyapunov-related workstreams. The strongest closed
loop so far appears to come from
\texttt{LyapunovSafetyFilterMPCTargetSelectorTermAblation.ipynb}, but that path
uses a richer target selector with several objective terms:
\begin{align}
&\|r_s-y_{\mathrm{sp}}\|_{Q_r}^2
+\|u_s-u_{\mathrm{applied},k}\|_{R_{u,\mathrm{ref}}}^2
+\|u_s-u_{s,\mathrm{prev}}\|_{R_{\Delta u,\mathrm{sel}}}^2 \nonumber \\
&\qquad
+\|x_s-x_{s,\mathrm{prev}}\|_{Q_{\Delta x}}^2
+\|x_s-\hat x_k\|_{Q_{x,\mathrm{ref}}}^2.
\label{eq:ablation_selector_objective}
\end{align}

Those terms are useful, and the ablation notebook was designed to measure their
closed-loop value. But they also make interpretation harder. When the target is
good, it is not immediately clear which mechanism caused the improvement:
output matching, current-input anchoring, previous-target smoothing, or the weak
observer-state anchor.

The direct frozen-output-disturbance path intentionally starts from a simpler
question:
\begin{quote}
If the output disturbance estimate is fixed, can a direct Lyapunov MPC track the
resulting steady target without relying on a separate candidate controller or a
large target-selector objective?
\end{quote}

That question is important because a final reportable controller should separate
three roles:
\begin{enumerate}[leftmargin=1.5em]
\item the model and disturbance estimate,
\item the steady target calculation,
\item the Lyapunov-constrained online controller.
\end{enumerate}
The current direct path isolates these roles better than the safety-filter
ablation path, even though the first hard/unbounded run is infeasible.

\section{Model and Coordinate Conventions}

The controller-side model is used in scaled deviation coordinates with an
output-disturbance-only augmentation:
\begin{align}
x_{k+1} &= A x_k + B u_k, \\
y_k &= Cx_k + d_k, \\
d_{k+1} &= d_k.
\end{align}

The augmented state is
\begin{equation}
z_k =
\begin{bmatrix}
x_k \\
d_k
\end{bmatrix},
\qquad
\hat z_k =
\begin{bmatrix}
\hat x_k \\
\hat d_k
\end{bmatrix}.
\end{equation}

Here \(x_k\) is the controller state in scaled deviation coordinates, \(u_k\) is
the manipulated input in scaled deviation coordinates, \(d_k\) is the estimated
output disturbance, and \(y_k\) is the controlled output in scaled deviation
coordinates. The notebook builds this model through
\texttt{augmentation\_mode="output\_disturbance"}. The implementation rejects
models with a nonzero disturbance-to-state block \(A_{xd}\), because the direct
target solver assumes that the frozen disturbance affects only the output
equation.

\section{Frozen Output-Disturbance Target}

\subsection{Unbounded Target}

At each time step \(k\), the target disturbance is fixed:
\begin{equation}
d_s = \hat d_k.
\end{equation}
Only \(x_s\) and \(u_s\) are solved. The steady-state equations are
\begin{align}
(I-A)x_s - Bu_s &= 0, \label{eq:ss_dyn}\\
Cx_s &= y_{\mathrm{sp},k} - \hat d_k. \label{eq:ss_out}
\end{align}

Stacking \eqref{eq:ss_dyn} and \eqref{eq:ss_out} gives
\begin{equation}
\underbrace{
\begin{bmatrix}
I-A & -B \\
C & 0
\end{bmatrix}}_{M}
\begin{bmatrix}
x_s \\
u_s
\end{bmatrix}
=
\begin{bmatrix}
0 \\
y_{\mathrm{sp},k}-\hat d_k
\end{bmatrix}.
\label{eq:target_stacked}
\end{equation}

The unbounded target solver in
\texttt{Lyapunov/frozen\_output\_disturbance\_target.py} solves
\eqref{eq:target_stacked} directly when possible and falls back to least squares
when needed. It reports the rank of \(M\), its condition number, singular-value
diagnostics, exact-vs-least-squares flags, residuals, and input-bound checks.

\subsection{Reduced Representation}

When \(I-A\) is invertible,
\begin{equation}
x_s = (I-A)^{-1}Bu_s,
\end{equation}
so the output equation can be written as
\begin{equation}
G u_s = y_{\mathrm{sp},k}-\hat d_k,
\qquad
G := C(I-A)^{-1}B.
\label{eq:reduced_target}
\end{equation}
The code uses this reduced form for diagnostics and optionally for bounded
least-squares calculations.

\subsection{Bounded Target}

The bounded mode keeps the same physics but enforces the input box:
\begin{equation}
u_{\min} \le u_s \le u_{\max}.
\end{equation}
If the exact target is outside the box, the bounded solve minimizes a residual
objective:
\begin{equation}
\min_{x_s,u_s}
\left\|(I-A)x_s-Bu_s\right\|_{Q_{\mathrm{dyn}}}^2
+
\left\|Cx_s-(y_{\mathrm{sp},k}-\hat d_k)\right\|_{Q_y}^2
\end{equation}
subject to the input bounds. This is the natural next target mode because the
first unbounded run showed exact target residuals but inadmissible steady
inputs.

\section{Direct Lyapunov MPC Formulation}

\subsection{Prediction Model}

Inside the MPC problem, the disturbance is frozen over the prediction horizon:
\begin{align}
x_{i+1|k} &= A x_{i|k}+B u_{i|k},\\
y_{i|k} &= Cx_{i|k}+\hat d_k,\\
x_{0|k} &= \hat x_k.
\end{align}

The steady target \((x_s,u_s,d_s,y_s)\) is computed first and then held fixed
while the online MPC is solved.

\subsection{Objective}

The direct controller now uses the normal output-tracking MPC objective:
\begin{align}
J_k =
&\sum_{i=0}^{N_p-1}
\|y_{i|k}-y_k^{\mathrm{track}}\|_Q^2
\nonumber\\
&+
\sum_{i=0}^{N_c-1}
\|\Delta u_{i|k}\|_R^2.
\label{eq:direct_objective}
\end{align}

With the default setting \texttt{use\_target\_output\_for\_tracking=False},
\(y_k^{\mathrm{track}}\) is the scheduled setpoint \(y_{\mathrm{sp},k}\), not
the admissible target output \(y_s\). The steady target variables \(x_s\) and
\(u_s\) are not objective anchors. They remain in the direct method only because
the Lyapunov contraction and terminal admissibility checks need a steady target
center.

\subsection{Terminal Ingredients}

The matrix \(P_x\) and local feedback \(K_x\) are inherited from the standard
tracking Lyapunov core. The terminal set condition is
\begin{equation}
(x_{N_p|k}-x_s)^\top P_x (x_{N_p|k}-x_s)
\le \alpha_k,
\end{equation}
where \(\alpha_k\) is computed from the terminal input-admissibility logic.

\subsection{Hard Lyapunov Contraction}

The tracking error is
\begin{equation}
e_{x,k}=\hat x_k-x_s,
\end{equation}
and the Lyapunov function is
\begin{equation}
V_k = e_{x,k}^\top P_x e_{x,k}.
\end{equation}
Hard mode enforces the one-step contraction inequality
\begin{equation}
(x_{1|k}-x_s)^\top P_x (x_{1|k}-x_s)
\le
\rho V_k + \varepsilon_{\mathrm{lyap}}.
\label{eq:hard_contraction}
\end{equation}

The saved run used \(\rho=0.98\) and
\(\varepsilon_{\mathrm{lyap}}=10^{-9}\). Since the target was outside the
admissible input box, \eqref{eq:hard_contraction} made the online problem
infeasible for every step.

\subsection{Soft Lyapunov Contraction}

Soft mode introduces one nonnegative slack:
\begin{align}
(x_{1|k}-x_s)^\top P_x (x_{1|k}-x_s)
&\le
\rho V_k + \varepsilon_{\mathrm{lyap}}+\sigma_k,\\
\sigma_k &\ge 0.
\end{align}
The objective becomes
\begin{equation}
J_k^{\mathrm{soft}} = J_k + \lambda_\sigma \sigma_k.
\end{equation}
The notebook exposes \(\lambda_\sigma=10^6\). Soft mode is not a way to hide a
bad target; it is a diagnostic and feasibility-relief mechanism that tells us
how much contraction must be relaxed.

\subsection{Interpretation}

The target solver is doing what it was asked to do. It finds an exact
unbounded steady target for every step. The problem is that the target is not
admissible for the constrained MPC. The step table records
\texttt{target\_exact\_within\_bounds=False} for all 1600 steps. For half the
steps both input-target components are beyond the upper side of the box; for
the other half both are beyond the lower side. In this situation the Lyapunov
center is physically unreachable under the online input constraints, so a hard
one-step contraction to that center is too strong.

The correct conclusion is not that direct Lyapunov MPC should be discarded. The
correct conclusion is that the next direct experiment must make the target
admissible or make contraction diagnostic rather than absolute:
\begin{enumerate}[leftmargin=1.5em]
\item run \texttt{target\_mode="bounded"} with \texttt{lyapunov\_mode="hard"},
\item run \texttt{target\_mode="bounded"} with \texttt{lyapunov\_mode="soft"},
\item inspect slack, residuals, and active input bounds,
\item only then decide whether a minimal input-anchor or previous-target
regularizer should be reintroduced.
\end{enumerate}

\section{Comparison With the Stronger Ablation Path}

The best current behavior has come from the safety-filter target-selector
ablation workflow, not from this first direct run. That is not surprising. The
ablation selector includes terms that act like practical guardrails:
\begin{itemize}[leftmargin=1.5em]
\item \textbf{Output target term}: keeps \(r_s\) or \(y_s\) near the scheduled
setpoint.
\item \textbf{Applied-input anchor}: discourages a target \(u_s\) that is far
from the current operating region.
\item \textbf{Previous-input smoothing}: suppresses target-input jumps.
\item \textbf{Previous-state smoothing}: suppresses target-state jumps.
\item \textbf{Weak \(\hat x_k\) anchor}: lets the target drift toward the
current estimated operating point without chasing every transient.
\end{itemize}

The direct path removes those terms so that the target is a pure frozen
output-disturbance steady solve. The prior run shows the cost of that
simplification: exact output matching can choose a steady input outside the
admissible box. In other words, the ablation path's ``too many objective terms''
were not just decorative. At least one of them, especially the applied-input
anchor or the bounded/admissible-target logic, was helping keep the Lyapunov
center usable.

The useful research direction is therefore not to copy the whole five-term
objective into the direct path. A cleaner sequence is:
\begin{enumerate}[leftmargin=1.5em]
\item first enforce admissibility with the bounded target mode,
\item then use soft Lyapunov mode to measure contraction pressure,
\item then add at most one or two target regularizers if the bounded target is
admissible but still too discontinuous or too far from the operating region.
\end{enumerate}

This would keep the direct controller reportable while preserving the empirical
lesson from the ablation notebook.

\section{Literature Context}

\subsection{Offset-Free MPC and Disturbance Models}

The output-disturbance augmentation used here is aligned with the offset-free
MPC literature. Muske and Badgwell describe the use of step disturbance models
and a steady-state target calculation to remove estimated disturbance effects in
linear MPC \cite{muske2002}. Pannocchia and Rawlings show that integrating
disturbances can deliver zero steady-state offset under general conditions and
that the disturbance dimension needed for offset-free behavior is tied to the
number of measured variables, not simply to the number of controlled variables
\cite{pannocchia2003}.

The present controller uses the simple output disturbance model
\[
y=Cx+d,\qquad d^+=d,
\]
and freezes \(d_s=\hat d_k\) inside the target calculation. That is a deliberate
research simplification. It lets the run answer whether the direct Lyapunov MPC
can work with the cleanest output-offset target before considering richer
disturbance models.

\subsection{Constrained MPC Stability}

The terminal admissibility constraint, Lyapunov function, and receding-horizon
structure follow the standard stability logic of constrained MPC. Mayne,
Rawlings, Rao, and Scokaert summarize how terminal penalties, terminal
constraints, and local stabilizing controllers provide stability and feasibility
structure for constrained MPC
\cite{mayne2000}. In this project, those ingredients are implemented through
\texttt{Lyapunov/lyapunov\_core.py}.

\subsection{Tracking MPC and Artificial Steady States}

The first direct run also points to a tracking-MPC issue: exact target matching
is not enough when the target is inadmissible. Limon, Alvarado, Alamo, and
Camacho propose tracking MPC for constrained linear systems with artificial
steady states and inputs; if the requested target is not admissible, the
controller can steer toward the closest admissible steady state \cite{limon2008}.
That idea is close in spirit to the next step for this repository: use bounded
target mode first, then consider whether a small artificial-reference penalty is
needed.

\subsection{Lyapunov MPC for Process Systems}

Lyapunov-based MPC has a long process-control motivation because chemical
plants combine nonlinear dynamics, input limits, state constraints, and
disturbances. Mhaskar, El-Farra, and Christofides develop Lyapunov-based
predictive control for nonlinear systems with state and input constraints,
including hard and soft constraint interpretations and chemical process examples
\cite{mhaskar2006}. The current direct path is simpler and linearized, but it
uses the same conceptual move: impose a Lyapunov decrease condition inside the
predictive controller.

\subsection{Predictive Safety Filters}

The older repository path is closer to predictive safety filtering: an upstream
controller proposes an input and a filter modifies it only if needed.
Wabersich and Zeilinger formalize predictive safety filters for learning-based
control, using MPC-style backup trajectories to ensure constraint satisfaction
for systems controlled by otherwise unconstrained learning policies
\cite{wabersich2021}. That literature supports the existing safety-filter
architecture. The direct path in this report is different: it is not a filter
around another policy, but a standalone Lyapunov MPC controller.

\section{Recommended Next Stage}

The next stage is to execute the rewritten notebook in the scientific Python
environment and inspect the saved four-case grid:
\begin{align}
&(\texttt{unbounded},\texttt{hard}),\quad
(\texttt{bounded},\texttt{hard}),\\
&(\texttt{unbounded},\texttt{soft}),\quad
(\texttt{bounded},\texttt{soft}).
\end{align}

The main success criteria should be:
\begin{itemize}[leftmargin=1.5em]
\item solver success rate,
\item target success rate,
\item exact and bounded target residuals,
\item active input-bound counts,
\item hard contraction rate,
\item soft slack mean, maximum, and active-step count,
\item output RMSE in physical units,
\item whether the controller uses target output or raw setpoint as the stage
tracking target.
\end{itemize}

The case to inspect first is:
\begin{quote}
\texttt{target\_mode="bounded"}, \texttt{lyapunov\_mode="soft"}
\end{quote}
with the same horizons and disturbance scenario. This case answers the most
important immediate question: after target admissibility is enforced, how much
Lyapunov slack is required to keep the problem feasible?

If bounded-soft succeeds but has large or frequent slack, the next tuning knobs
should be:
\begin{enumerate}[leftmargin=1.5em]
\item relax \(\rho\) slightly, for example from \(0.98\) toward \(0.995\),
\item increase horizon or reduce terminal aggressiveness,
\item test \texttt{use\_target\_output\_for\_tracking=True},
\item add only a minimal target regularizer, preferably an applied-input anchor
or previous-target smoothing, rather than restoring all five ablation terms.
\end{enumerate}

\section{Conclusion}

The direct frozen-output-disturbance Lyapunov MPC path is the right structure
for a clean mathematical report: one disturbance model, one target calculation,
one Lyapunov-constrained controller, and one debug/export schema. The prior run
does not yet demonstrate closed-loop control performance, but it does clearly
identify the next technical bottleneck. The unbounded target calculation is
exact but inadmissible, and the hard Lyapunov contraction makes the online MPC
infeasible at every step.

The next state should therefore be a bounded-target and soft-contraction study.
That will tell us whether the direct architecture can become competitive with
the stronger safety-filter ablation path without inheriting all of its target
selector objective complexity.
